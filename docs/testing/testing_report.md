# Отчет по тестированию ACCAC

## Область проверки

Тестирование охватывает:

- доступность подключения к PostgreSQL;
- наличие и корректность данных интерфейсного макета;
- ссылочную целостность связей антенн;
- нагрузочные сценарии чтения данных из `tb_CMP` и `tb_ANT`.

## Инструменты

- Apache JMeter
- JDBC-подключение к PostgreSQL
- CSV-параметризация идентификаторов компонентов и антенн

## Тестовые артефакты

- Основной план: `tests/jmeter/plans/ACCAC.jmx`
- План поиска лимитов: `tests/jmeter/plans/ACCAC_DB_LIMITS.jmx`
- CSV-данные: `tests/jmeter/data/cmp_ids.csv`, `tests/jmeter/data/ant_ids.csv`
- Скрипт поиска лимитов: `tests/jmeter/run_accac_db_limits.sh`

## Порядок проведения

1. Подготовить БД через `db/postgresql/run_all.sh`.
2. Проверить локальный конфиг приложения и параметры подключения JMeter.
3. Запустить основной JMeter-план для функциональной проверки и базовой нагрузки.
4. При необходимости запустить `tests/jmeter/run_accac_db_limits.sh` для поиска предельного числа потоков.
5. Сохранить локальные артефакты в `tests/jmeter/jmeter-results/` без коммита в репозиторий.
